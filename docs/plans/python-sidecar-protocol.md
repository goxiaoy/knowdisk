# Python Sidecar Protocol

The Bun main process and Python worker communicate over `stdio` using line-delimited JSON.

Each line is exactly one JSON frame.

## Frame types

### Request

```json
{
  "id": "req-1",
  "method": "index_node",
  "params": {
    "nodeId": "node-1"
  }
}
```

### Response

Success:

```json
{
  "id": "req-1",
  "result": {
    "ok": true
  }
}
```

Failure:

```json
{
  "id": "req-1",
  "error": {
    "code": "WORKER_ERROR",
    "message": "Failed to index node"
  }
}
```

### Event

```json
{
  "type": "index_status_changed",
  "payload": {
    "phase": "indexing",
    "queueDepth": 1
  }
}
```

## Rules

- request frames must contain `id`, `method`, and `params`
- response frames must contain `id` and exactly one of `result` or `error`
- event frames must contain `type` and `payload`
- frame `id` values are generated by the caller and echoed by the responder
- all frames must fit on one line
